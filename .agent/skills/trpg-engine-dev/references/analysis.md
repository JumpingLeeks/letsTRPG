# AI TRPG 시스템 개발 사후 분석 리포트 (Post-Mortem)

## 1. 개요
본 리포트는 실시간 멀티플레이어 AI TRPG 시스템 구축 과정에서 발생한 주요 기술적 장애와 그 해결 과정을 기록하여, 향후 유사 프로젝트의 안정성을 확보하는 데 목적이 있습니다.

## 2. 주요 실패 지점 (Failure Points)

### A. 패키지 의존성 및 임포트 구조 오류
- **문제**: 표준 `langchain` 라이브러리와 사용자의 특수한 `langchain-classic` 패키지 사이의 구조적 차이를 간과함.
- **교훈**: `pip list`를 통한 환경 선분석 필수.

### B. 파이썬 실행 환경(venv) 불일치
- **문제**: 시스템 전역 `uvicorn` 실행으로 인한 충돌.
- **교훈**: `.\venv\Scripts\python.exe` 명시적 호출 필수.

### C. 비동기 로직의 중복 및 간섭
- **문제**: `check_timeout`과 루프 간의 중복 브로드캐스트.
- **교훈**: 상태 변경과 전파 로직의 엄격한 분리 (advance_turn_if_needed).

### D. 함수 정의 순서
- **문제**: Tool 정의 전 참조 오류.
- **교훈**: 하향식 파싱을 고려한 소스코드 배치 최적화.
